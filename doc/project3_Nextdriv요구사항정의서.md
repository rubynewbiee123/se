## 1. 개요(Introduction)

### 1.1 문서의 목적

본 문서는 조직 내 파일을 중앙에서 관리하고 공유하는 클라우드 시스템 **NextDriv**의 **요구사항 정의서(Requirements Definition Document, RDD)**이다.

본 문서는 시스템이 제공해야 할 기능적·비기능적·인터페이스 요구사항을 식별하고 정의한다. 본 문서의 결과물은 이후 요구사항 분석서(SRS), 소프트웨어 설계서, 테스트 계획서의 기준이 된다.

본 문서의 적용 범위는 다음과 같다.

- NextDriv가 수행해야 할 기능 명세
- 200명 동시 접속 환경에서 충족해야 할 비기능 명세
- 사용자, 외부 시스템, 스토리지 백엔드와의 인터페이스 명세
- 프로젝트 II에서 식별한 7개 품질 요소 기반 품질 요구사항
- 사내 정책 및 법령에 따른 제약사항

### 1.2 프로젝트 개요

#### 1.2.1 프로젝트 정의

NextDriv는 약 200명 규모의 조직 구성원이 동시 사용하는 웹 기반 클라우드 파일 관리 시스템이다. 사용자는 웹 브라우저로 파일을 업로드·다운로드하고, 폴더로 정리하며, 권한에 따라 팀원과 공유한다. 시스템은 파일 버전 관리와 동시 접근 제어를 통해 협업 과정의 데이터 손실을 방지한다.

#### 1.2.2 주요 기능 설명

| ID | 기능 | 설명 |
| :--- | :--- | :--- |
| F01 | 파일 업로드/다운로드 | 다양한 형식의 파일을 웹 브라우저에서 업로드하고 다운로드한다. |
| F02 | 폴더 관리 | 폴더를 생성·이름 변경·삭제하며 다단계 구조를 지원한다. |
| F03 | 파일 공유 및 권한 관리 | 사용자별 열람·수정·댓글 권한을 부여하고 외부 공유 링크를 제공한다. |
| F04 | 파일 검색 | 파일명·날짜·업로더·파일 유형 기준으로 검색한다. |
| F05 | 파일 버전 관리 | 수정 이력을 보관하고 이전 버전을 조회·복원한다. |
| F06 | 사용자 계정 및 권한 관리 | 관리자가 계정을 생성·삭제하고 저장 공간과 그룹을 관리한다. |
| F07 | 동시 접근 제어 | 파일 잠금과 충돌 처리를 제공한다. |
| F08 | 휴지통 및 복구 | 삭제된 파일을 일정 기간 보관하고 복구한다. |
